# Рабочее пространство для написания книг с AI

Структурированное рабочее пространство для совместного написания книг между автором-человеком и AI-агентами. Поддерживает несколько языков через независимые языковые домены (`books/{iso}/`).

Другие языки: [English](README.md) · [Português](README.pt.md)

---

## Что это такое

Этот каркас даёт тебе и AI-агенту общее организованное пространство для написания книги — с чёткими правилами о том, кто чем владеет, как черновики движутся к канону и как агент сохраняет контекст между главами, не перечитывая всю рукопись каждый раз.

**Основная идея:** три пространства со строгим приоритетом.

```
USER_DRAFT  →  AGENT_DRAFT  →  APPROVED
(твой сырой ввод)  (агент интерпретирует)  (утверждённый канон)
```

Агент никогда не трогает твои черновики. Ты никогда не пишешь вручную интерпретации агента. Только ты переносишь контент в APPROVED.

---

## Структура репозитория

```
books/
├── ru/                        # Русский языковой домен
│   ├── CONTEXT/
│   │   ├── CONVENTIONS/       # Правила поведения агента для этого языка
│   │   ├── MEMORY/            # Универсальные предпочтения автора (все книги)
│   │   └── RESEARCH/          # Исследования по мастерству (структура, размер глав, жанры...)
│   ├── _template/             # Скопировать для создания новой книги
│   └── {bookName}/            # Конкретная книга
│
├── en/                        # Английский языковой домен
│   ├── CONTEXT/
│   ├── _template/
│   └── {bookName}/
│
└── pt/                        # Португальский языковой домен
    ├── CONTEXT/
    ├── _template/
    └── {bookName}/
```

Каждый языковой домен независим. Папка `_template/` — заготовка для новой книги.

## Быстрый старт

### 1. Создай новую книгу

Скопируй шаблон для своего языка:
```
books/ru/_template/  →  books/ru/моя-книга/
```

### 2. Заполни концептуальные файлы

Начни с этих файлов в папке новой книги — пиши свободно, формат не обязателен:

| Файл | Что туда |
|------|---------|
| `USER_DRAFT/idea.md` | Сырой замысел — поток сознания, образы, вопросы |
| `BRIEF.md` | Жанр, аудитория, суть, тема (максимум одна страница) |
| `PLAN.md` | Поглавный план и арки персонажей |
| `USER_DRAFT/universe/world.md` | Что это за мир |
| `USER_DRAFT/universe/characters/{имя}.md` | Каждый персонаж — кто он, чего хочет |

### 3. Попроси агента интерпретировать

Укажи агенту на `CLAUDE.md` книги и попроси интерпретировать один элемент за раз. Например:
- «Интерпретируй `USER_DRAFT/idea.md` в трёх вариантах»
- «Напиши три версии главы 001 на основе моего черновика»

Агент всегда создаёт **минимум 3 варианта** (v1, v2, v3). Каждый вариант — осмысленно другой подход, а не косметическое различие.

### 4. Просмотри и утверди

Прочитай варианты. Когда будешь готов:
- Скажи агенту, какой вариант предпочитаешь (или что объединить)
- Перенеси выбранный контент в `APPROVED/` сам — агент не делает это без твоей команды

### 5. Продолжай писать

Для каждой новой главы агент читает:
1. Резюме предыдущей утверждённой главы (`APPROVED/chapters/N.short.md`)
2. Снимок состояния из последней главы (`state-snapshot.md`)
3. Утверждённые факты вселенной (`APPROVED/universe/`)
4. Нарративный план (`PLAN.md`)

Именно так агент поддерживает контекст, не перечитывая всю рукопись.

## Три пространства

### USER_DRAFT — твой сырой ввод
Всё, что ты пишешь, сначала попадает сюда. Агент это читает, но **никогда не редактирует**. Пиши как угодно: обрывками, с противоречиями, с вопросами к себе. Агент будет интерпретировать, а не исправлять.

### AGENT_DRAFT — интерпретации агента
Агент создаёт минимум 3 варианта для каждой главы или элемента вселенной. Каждая папка варианта содержит:
- `content.md` — полный текст
- `short.md` — резюме ~500 слов для повторного использования как контекста
- `state-snapshot.md` — состояние мира после этой главы (местонахождение персонажей, эмоции, открытые нити)

Варианты должны предлагать **реальный выбор** — разный темп, точку зрения, тональность — а не мелкие отличия в формулировках.

### APPROVED — твой канон
Высший приоритет. Только ты переносишь контент сюда. Когда что-то утверждено, агент считает это неизменным фактом. При написании любой новой главы агент читает APPROVED и игнорирует AGENT_DRAFT.

**Правило приоритета:** `APPROVED > USER_DRAFT > AGENT_DRAFT`

Если агент находит противоречие между пространствами — он записывает его в `STATUS.md` и ждёт твоего решения. Никогда не разрешает противоречия самостоятельно.

## Управление состоянием

Роман слишком длинный, чтобы поместиться в контекст. Каркас решает это двумя слоями:

**Статический слой — `APPROVED/universe/`**
Постоянные факты, которые меняются редко: биографии персонажей, правила мира, история до начала книги.

**Динамический слой — `state-snapshot.md`**
Состояние мира *после* каждой главы: где каждый персонаж, что он чувствует, что изменилось, какие нарративные нити открыты. Агент заполняет это после каждого варианта.

Перед написанием главы N агент читает:
- `APPROVED/chapters/{N-1}.short.md` — что произошло
- `state-snapshot.md` главы N-1 — текущее состояние
- `APPROVED/universe/` — постоянные факты

Этого достаточно для связного написания без перечитывания всего.

## Отслеживание работы

**`STATUS.md`** — рабочий журнал агента. Обновляется после каждого действия. Содержит:
- Текущую задачу
- Фазу каждой главы и элемента вселенной (USER_DRAFT / AGENT_DRAFT / APPROVED)
- Вопросы, ожидающие твоего решения

**`FEEDBACK.md`** — журнал отклонённых вариантов. Когда ты отклоняешь вариант, агент записывает что не сработало и почему. Предотвращает повторение тех же ошибок.

**`MEMORY.md`** — твои предпочтения для конкретной книги (тональные ориентиры, что избегать, что ты одобрял).

## Стили и несколько версий выпуска

Книга может быть выпущена в нескольких нарративных стилях — тот же сюжет и факты, другой голос и тональность.

Стили живут в `APPROVED/universe/styles/`. Каждый утверждённый стиль порождает отдельную папку `RELEASES/{styleName}/` с полным текстом книги, переписанным в этом голосе.

Примеры стилей: `main` (основной), `romantic` (романтический), `gothic` (готический), `goblin` (гоблинский).

**Стиль меняет подачу, но не содержание.** Агент никогда не меняет сюжет, персонажей или факты при применении стиля.

## Языковые домены

Каждый `books/{iso}/` независим. Добавление нового языка означает создание нового домена со своим `CONTEXT/` (конвенции и исследования на этом языке) и своим `_template/`.

Доступные домены:
- `books/ru/` — Русский (активен, включает демо-книгу `mur/`)
- `books/en/` — English (активен, включает демо-книгу `bess/`)
- `books/pt/` — Português (активен, включает демо-книгу `lua/`)

---

## Точки входа агента

Агент читает два файла CLAUDE.md в начале любой сессии:

1. **`/CLAUDE.md`** (корень) — глобальная архитектура, правила приоритета, протокол управления состоянием
2. **`books/{iso}/{book}/CLAUDE.md`** — контекст конкретной книги, специальные правила, быстрые ссылки

Перед написанием главы агент также читает BRIEF.md, PLAN.md, MEMORY.md и FEEDBACK.md книги.

| Создать новую книгу | `books/{iso}/_template/` |
| Посмотреть правила поведения агента | `books/{iso}/CONTEXT/CONVENTIONS/index.md` |
| Отследить прогресс работы | `{book}/STATUS.md` |
| Прочитать нарративный план | `{book}/PLAN.md` |
| Прочитать концепцию + тему | `{book}/BRIEF.md` |
| Найти утверждённые факты персонажей | `{book}/APPROVED/universe/characters/` |
| Найти утверждённые правила мира | `{book}/APPROVED/universe/lore.md` |
| Посмотреть текущее состояние после последней главы | `state-snapshot.md` последней утверждённой главы |
| Записать причину отклонения варианта | `{book}/FEEDBACK.md` |
| Прочитать исследования по мастерству | `books/{iso}/CONTEXT/RESEARCH/` |
