# FEEDBACK — Variantes Rejeitadas

Este ficheiro é preenchido pelo agente sempre que o autor rejeita uma variante.
Objectivo — não repetir os mesmos erros nas próximas variantes.

---

## Formato de Entrada

```
### [Data] · [O que foi rejeitado]
**Razão:** [o que especificamente não funcionou]
**Conclusão para o agente:** [o que evitar na próxima vez]
```
