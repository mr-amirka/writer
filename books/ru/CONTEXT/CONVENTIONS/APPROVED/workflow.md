## CONV-007 · Работа с вариантами
`#CONV-007` · claude-sonnet-4-6 · 2026-06-05 12:00 · `accepted`

При создании v1 / v2 / v3 для каждой главы или крупного элемента вселенной:

| Вариант | Фокус |
|---------|-------|
| v1 | Наиболее близкий к USER_DRAFT — сохраняет авторскую интонацию, минимум отклонений |
| v2 | Структурная переработка — иной темп, другая точка зрения или порядок событий |
| v3 | Смелый эксперимент — другой тон, нестандартное начало, вынос в другой жанровый регистр |

Не делать варианты формально разными — каждый должен предлагать реальный, значимый выбор.

**Каждый вариант включает три файла:** `content.md` + `short.md` + `state-snapshot.md`.

## CONV-008 · Протокол перехода между главами
`#CONV-008` · claude-sonnet-4-6 · 2026-06-05 12:00 · `accepted`

Обязательно перед написанием главы N:
1. `APPROVED/chapters/{N-1}.short.md` — что произошло
2. `state-snapshot.md` утверждённого варианта главы N-1 — где все, что у всех
3. `PLAN.md` — что должно произойти в N
4. `BRIEF.md` — тема, мораль и физические якоря (раздел «Тема и мораль»)
5. `APPROVED/universe/characters/` — актуальные анкеты (статика)
6. `APPROVED/universe/world.md` + `lore.md` — правила мира
7. `MEMORY.md` — предпочтения автора по этой книге
8. `FEEDBACK.md` — что отвергалось ранее и почему

Не начинать главу без этого контекста.

**Статика vs. динамика:**
- `APPROVED/universe/` — постоянные факты. Меняются редко, только по инструкции автора.
- `state-snapshot.md` — текущее состояние после последней главы. Заполняется агентом после каждого варианта.

## CONV-011 · Что агент НЕ делает без явной инструкции
`#CONV-011` · claude-sonnet-4-6 · 2026-06-05 12:00 · `accepted`

- Не редактирует `USER_DRAFT/`.
- Не переносит контент в `APPROVED/` — это делает только автор.
- Не удаляет старые варианты из `AGENT_DRAFT/`.
- Не придумывает новых персонажей или фактов мира вне `APPROVED/universe/`.
- Не меняет уже одобренные факты из `APPROVED/universe/`.
- Не обновляет `APPROVED/universe/` самостоятельно — только по явной инструкции автора.
- Не записывает текущее состояние персонажей в `APPROVED/universe/` — только в `state-snapshot.md`.

## CONV-013 · Файлы статуса и темы
`#CONV-013` · claude-sonnet-4-6 · 2026-06-05 12:00 · `accepted`

**`STATUS.md`** — трекер работы агента. Обновляется при каждом изменении статуса. Не путать с PLAN.md.

**`BRIEF.md`** — одностраничный документ книги. Включает **тему, мораль, анти-мораль и физические якоря**. Агент читает перед каждой задачей.

**`FEEDBACK.md`** — причины отклонения вариантов. Предотвращает повторение ошибок.

**Лор для канона** — раздел `## Лор для канона` каждого `state-snapshot.md`. Автор решает, что переносить в `APPROVED/universe/`.

## CONV-016 · Git-коммиты
`#CONV-016` · claude-sonnet-4-6 · 2026-06-06 · `accepted`

Все коммиты выполняются только от имени владельца. Никакой атрибуции агентов — ни `Co-Authored-By:`, ни ID моделей, ни подписей AI-инструментов в сообщениях коммитов или метаданных.

История коммитов — это запись автора, не агента.

## CONV-016-A · Предложение сообщения коммита
`#CONV-016-A` · claude-sonnet-4-6 · 2026-06-06 · `accepted`

Перед любым коммитом агент обязан предложить сообщение коммита и дождаться подтверждения владельца. Никогда не коммитить молча или немедленно.

Формат предложения:
```
Предлагаемое сообщение коммита:
> {сообщение}

Закоммитить?
```

Коммитить только после явного подтверждения. Если владелец изменил сообщение — использовать его версию дословно.
