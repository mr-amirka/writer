## CONV-006 · Формат маркера
`#CONV-006` · claude-sonnet-4-6 · 2026-06-06 15:30 · `accepted`

Каждый пункт в каждом файле (кроме RELEASES и аналогичных чистовых файлов) несёт маркер:

```
`#CONV-NNN` · {agent-id} · YYYY-MM-DD HH:MM · `{status}`
```

- **ID** — глобально уникальный, последовательный. Префикс: `CONV-` универсальный, `CONV-RU-` / `CONV-EN-` / `CONV-PT-` доменный.
- **Agent** — ID модели, предложившей пункт, например `claude-sonnet-4-6`.
- **Timestamp** — с точностью до минуты, ISO-8601.
- **Status** — одно из: `draft` · `accepted` · `rejected`.

При принятии или отклонении пункта поле статуса обновляется на месте. Исходный текст **никогда не удаляется** — меняется только поле статуса.

## CONV-006-A · Формат вопроса
`#CONV-006-A` · claude-sonnet-4-6 · 2026-06-06 15:30 · `accepted`

Когда решение требует участия автора, агент ставит маркированный вопрос:

```
`?CONV-NNN-Q1` · {agent-id} · YYYY-MM-DD HH:MM

**Текст вопроса?**

→ **A:** вариант A
→ **B:** вариант B
→ **C:** вариант C

↳ *Ожидает автора*
```

Когда автор отвечает, ответ **дописывается ниже** вариантов — варианты не удаляются:

```
↳ *Ожидает автора*

↳ **Автор · YYYY-MM-DD HH:MM:** B — потому что...
```

## CONV-006-B · Область применения маркеров
`#CONV-006-B` · claude-sonnet-4-6 · 2026-06-06 15:30 · `accepted`

**Файлы, требующие маркеров:** все `.md` файлы в репозитории, кроме:
- `RELEASES/*/index.md` и аналогичных чистовых файлов (текст книги для читателей)
- `RELEASES/*/metadata.md`
- `README.md`, `README.ru.md`, `README.pt.md` (публичные документы)

**Файлы, требующие маркеров обязательно:** CONVENTIONS/, CONTEXT/, STATUS.md, BRIEF.md, PLAN.md, MEMORY.md, FEEDBACK.md, APPROVED/universe/*, state-snapshot.md.

## Открытые вопросы

`?CONV-006-Q1` · claude-sonnet-4-6 · 2026-06-06 15:30

**Применять ли маркеры ретроактивно ко всем существующим файлам или только к файлам, создаваемым или редактируемым в будущем?**

→ **A:** Ретроактивно — все существующие файлы получают маркеры за один проход сейчас (~100+ файлов).
→ **B:** Только вперёд — маркеры применяются к новым файлам и активно редактируемым; существующие не трогаются без другой причины.
→ **C:** Гибридно — CONVENTIONS и мета-файлы (STATUS, BRIEF, PLAN, MEMORY, FEEDBACK, state-snapshot, universe) маркируются сейчас; текстовые файлы глав — только при добавлении новых.

↳ *Ожидает автора*

↳ **Автор · 2026-06-06:** C — маркировать CONVENTIONS и мета-файлы сейчас; текстовые файлы глав — только при добавлении новых глав.

`?CONV-006-Q2` · claude-sonnet-4-6 · 2026-06-06 15:30

**Пространство ID: должны ли ID быть глобально уникальными для всех доменов, или ограниченными доменом?**

→ **A:** Глобальные — `CONV-001` означает одно правило в `books/ru/`, `books/en/`, `books/pt/`. Доменные правила получают префикс: `CONV-RU-001`.
→ **B:** Доменные — у каждого домена свой счётчик с 001; межданные ссылки используют полный путь.

↳ *Ожидает автора*

↳ **Автор · 2026-06-06:** A — глобальные ID, единое пространство для всех доменов.
