# RELEASES — Финальные сборки

Собранные версии книги для публикации или передачи. Каждая папка — отдельный нарративный стиль.

**Агент собирает RELEASE по запросу автора, взяв `APPROVED/chapters/` и применив стиль из `UNIVERSE/styles/`.**

---

## Структура

```
RELEASES/
├── main/                  # Основной стиль (styles/main.md) — полный текст
│   └── index.md
├── romantic/              # Стиль: styles/romantic.md
│   ├── index.md           # Полный текст книги в этом стиле
│   └── metadata.md        # Что изменилось относительно main
├── goblin/                # Стиль: styles/goblin.md
│   ├── index.md
│   └── metadata.md
└── {styleName}/           # Любой другой стиль
    ├── index.md
    └── metadata.md
```

**Принцип именования:** папка называется так же, как файл стиля в `APPROVED/universe/styles/`. Никаких `raw`, `default` или других псевдонимов.

**Содержимое index.md:** полный текст книги, не ссылки. Читатель должен видеть итоговый результат без необходимости открывать другие файлы.

---

## Процесс сборки RELEASE

1. Убедиться, что все нужные главы есть в `APPROVED/`.
2. Загрузить стиль из `UNIVERSE/styles/{styleName}.md`.
3. Для каждой утверждённой главы: переписать в голосе стиля, сохраняя факты и сюжет.
4. Собрать: пролог + главы по порядку + эпилог → `RELEASES/{styleName}/index.md`.
5. Заполнить `metadata.md`.

**Стиль не меняет сюжет, персонажей и факты — только способ подачи.**
